  5. Debouncing in JS.

    6. Ans. Debouncing in JavaScript is a practice used to improve browser performance. There might be some functionality in a web page which requires time-consuming computations. If such a method is invoked frequently, it might greatly affect the performance of the browser, as JavaScript is a single threaded language. Debouncing is a programming practice used to ensure that time-consuming tasks do not fire so often, that it stalls the performance of the web page. In other words, it limits the rate at which a function gets invoked.

    
        
        
            let counter = 0;
            function getData()
                {
                    console.log("Fetching Data" + counter++);
                }

            function myDebounce(call, d)
            {
                let timer;
                return function(...args)
                {
                    if(timer)
                    clearTimeout(timer);
                    timer = setTimeout(()=>
                    {
                        call();
                    },d);
                }
            }    

            const BetterFunction = myDebounce(getData, 2000);
  6. What is Throttling in JavaScript?

    7. Ans. Throttling implies limiting the number of times a function gets called in a certain time period. It will prohibit a function from executing if we have invoked it “recently.” Throttling also guarantees that a function runs at a consistent rate.

    
        

        const myThrottle = (fn, d) =>
        {
            return function(...args)
            {
                document.getElementById("myId").disabled = true;
                setTimeout(()=>
                {
                    fn();
                }, d);
            }
        }
        const newFun = myThrottle(()=>
        {
            document.getElementById("myId").disabled = false;
            console.log("User Clicked..!!");
        }, 2000)
  7. Difference between throttling and Debouncing.

    8. Ans. It's simple. They do the exact same thing (rate limiting) except while throttle is being called it'll fire your function periodically, while debounce only fires once at the end. Example: If you're scrolling, throttle will slowly call your function while you scroll (every X milliseconds).

  8. What is callback function in JS?

    9. Ans. A callback function is a function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action.

    
        const datas = [{name:"AB", occupation:"SDE"}, {name:"AC", occupation:"SDE1"}, {name:"AD", occupation:"SDE2"}];
        function getDatas()
        {
            setTimeout(()=>{
                let output = "";
                datas.forEach((data, index)=>{
                    output+= `${data.name} ${data.occupation} ${index}`
                })
                document.body.innerHTML = output;
            },1000)
        }
        function createData(newData, callback)
        {
            setTimeout(()=>{datas.push(newData); callback();},9000)
            
        }
        createData({name:"AS", occupation:"SDE3"}, getDatas);
  9. What is JS Promises?

    10. Ans. A promise is an object that may produce a single value some time in the future: either a resolved value, or a reason that it's not resolved (e.g a network error occured). A promise may be in one of 3 possible states: fulfilled, rejected or pending. A promise acts as a container for future value, like if you order any food from any site to deleiver it to your place that order record will be the promise and the food will be the value of that promise. then: When a promise is successful, we can then use the resolved data. catch: When a promise fails, we catch the error, and do something with the error. 

    
       let promiseToCleanTheRoom = new Promise(function(resolve, reject)
       {
        let isClean = true;
        if(isClean)
        {
            resolve(`clean.`);
        }
        else
        {
            reject(`not clean.`);
        }
       });
       promiseToCleanTheRoom.then(function(fromResolve){console.log(`The room is ${fromResolve}`)}).catch(function(fromReject){console.log(`The room is ${fromReject}`)});
